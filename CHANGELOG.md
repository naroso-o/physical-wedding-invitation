# 청첩장 전자책 뷰어 - 수정 완료 🎉

## ✅ 수정된 사항

### 1. 마지막 페이지 버그 수정

- **문제**: 마지막 페이지까지 넘겼을 때 cover2가 아닌 inside1이 보이던 문제
- **해결**: 페이지 구조를 완전히 재설계하여 4개의 독립 페이지로 변경
  - 기존: 2개의 양면 페이지 (front/back 구조)
  - 수정: 4개의 단면 페이지 (cover1 → inside1 → inside2 → cover2)

### 2. QR 코드 클릭 리다이렉트 추가

- **기능**: inside1의 QR 코드 영역을 클릭하면 모바일 청첩장으로 이동
- **설정 방법**:

```javascript
// index.html의 47번째 줄
const qrRedirectUrl = "https://example.com/mobile-invitation";
// ↑ 실제 모바일 청첩장 URL로 변경하세요
```

- **QR 위치 조정**: CSS에서 .qr-clickable 클래스 수정

```css
.qr-clickable {
  width: 40%; /* QR 영역 너비 */
  height: 40%; /* QR 영역 높이 */
  top: 50%; /* 상하 위치 (기본: 중앙) */
  left: 50%; /* 좌우 위치 (기본: 중앙) */
}
```

### 3. 데스크탑 가운데 정렬 개선

- **문제**: 한 페이지에서 두 페이지로 전환 시 정렬이 깨지던 문제
- **해결**:
  - 첫 페이지(cover1): 한 페이지만 표시, 화면 정중앙 배치
  - 중간 페이지(inside1+inside2): 두 페이지 펼쳐서 표시, 자동 가운데 정렬
  - 마지막 페이지(cover2): 한 페이지만 표시, 화면 정중앙 배치
  - 창 크기 변경 시 자동 재정렬

## 🎨 개선된 기능

### 페이지 전환 애니메이션

- 부드러운 슬라이드 인/아웃 효과 추가
- 자연스러운 페이지 전환 경험

### 반응형 레이아웃

- 모바일: 항상 한 페이지씩 표시
- 데스크탑: 중간 페이지는 펼쳐서 2페이지 표시
- 창 크기 변경에 즉각 대응

## 📱 사용 방법

### QR 코드 클릭 영역 확인하기

QR 영역을 시각적으로 확인하려면 CSS 디버그 옵션을 활성화하세요:

```css
/* index.html의 CSS 섹션에서 주석 해제 */
.qr-clickable:hover {
  background: rgba(255, 0, 0, 0.2); /* 마우스 올릴 때 빨간색 반투명 */
}
```

### 페이지 순서

1. **페이지 1**: cover1 (앞표지) - 단독 표시
2. **페이지 2**: inside1 (QR 코드 포함) - 데스크탑에서는 inside2와 함께 표시
3. **페이지 3**: inside2 - 데스크탑에서는 inside1과 함께 표시
4. **페이지 4**: cover2 (뒷표지) - 단독 표시

## 🔧 추가 커스터마이징

### QR 코드가 다른 위치에 있는 경우

QR 코드가 중앙이 아닌 다른 곳에 있다면:

```css
.qr-clickable {
  /* 예: 오른쪽 하단 */
  top: 70%;
  left: 70%;

  /* 예: 왼쪽 상단 */
  top: 30%;
  left: 30%;
}
```

### 페이지 크기 변경

```css
:root {
  --page-width: min(90vw, 400px); /* 모바일: 화면의 90%, 최대 400px */
}

@media (min-width: 768px) {
  :root {
    --page-width: 350px; /* 데스크탑: 고정 350px */
  }
}
```
